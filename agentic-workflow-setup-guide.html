Setup — Five Steps

1 Install Claude Code Terminal

The engine of the whole workflow — Claude that lives in your terminal and editor.

Open the Terminal app (press Cmd+Space, type "Terminal", hit Enter) and paste:

curl -fsSL https://claude.ai/install.sh | bash

Then start it and sign in:

claude
🖼️ What you'll see: a browser tab opens asking you to log in. Use your existing paid Claude account. Approve, return to Terminal — you're in.
⚠️ Don't set ANTHROPIC_API_KEY. If that environment variable exists, Claude Code bills per-token instead of using your subscription. Leave it unset.
💡 If claude says "command not found", run:
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc then source ~/.zshrc
  1. Ran the install command
  2. Ran claude and logged in via browser
  3. Saw the Claude Code prompt in Terminal

2 Connect Claude Code to VS Code Editor

So Claude's edits show up as live diffs right inside your editor.

  1. Open VS Code → Extensions panel (Cmd+Shift+X)
  2. Search "Claude Code" (by Anthropic) → Install
  3. Open your project folder in VS Code (File → Open Folder)
  4. Open the integrated terminal (Ctrl+`) and type claude
🖼️ What you'll see: a Claude panel docked in VS Code. When Claude edits a file, you get a side-by-side diff — green for added lines, red for removed — that you accept or reject.
💡 Useful moves: @ a filename to point Claude at it · hover a past message to rewind code to an earlier checkpoint · run claude --resume to pick up a previous conversation.

3 Wire up GitHub Version control

Where your code lives, and where pull requests happen.

3a — Install & log in to the GitHub CLI:

brew install gh
gh auth login
🖼️ Choose GitHub.com → HTTPS → Login with a web browser. Copy the one-time code shown, paste it in the browser, authorize. Done.

3b — Let Claude act on your repo (the agentic part): inside a claude session, run:

/install-github-app
🖼️ This walks you through installing the Claude GitHub app on your repository and setting up the secret it needs. After this, you can comment @claude fix the failing test on any issue or PR and Claude will open a session and push a fix.
⚠️ You must be an admin of the repo to install the app. If /install-github-app stalls, install manually at github.com/apps/claude.
  1. gh auth login succeeded
  2. Ran /install-github-app and selected your repo
  3. Tested @claude on a test issue

4 Bring in Slack Notify + trigger

Two separate apps do two different jobs — install both.

4a — Claude in Slack (kick off coding tasks by chatting):

  1. Go to the Claude on Slack marketplace page → Add to Slack
  2. Authorize it for your workspace
  3. In a channel, type /invite @Claude
  4. Try it: @Claude add a README to my repo
🖼️ Claude replies in-thread with status updates, then shows View Session and Create PR buttons when done.

4b — GitHub for Slack (get notified of PRs & CI):

  1. Install the official GitHub app from the Slack marketplace
  2. In your chosen channel run: /github subscribe your-username/your-repo
💡 Think of it as: Claude app = do the work, GitHub app = tell me what happened.
⚠️ Claude-in-Slack needs Claude Code on the web enabled. On a Team/Enterprise workspace an admin must turn it on under Settings → Claude Code. On a personal workspace your individual plan governs access.

5 Run the loop daily Workflow

Your repeatable rhythm once everything's connected.

  1. Start a branch: tell Claude "start a new branch for the login feature"
  2. Build: describe what you want; review the diffs; accept
  3. Commit: "commit this with a clear message" — Claude writes it and runs git commit
  4. Open a PR: "push and open a pull request" — Claude uses gh to do it
  5. Notify: GitHub app posts the PR to Slack automatically
  6. Iterate from anywhere: comment @claude address the review comments on the PR, or message @Claude in Slack
🎓 Returning-dev tip: you rarely type git commands yourself anymore. Describe the intent ("undo my last commit but keep the changes") and let Claude translate it — but read what it proposes before accepting.
